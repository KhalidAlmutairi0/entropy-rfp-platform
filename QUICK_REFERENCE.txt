BUG #1: CORS BROKEN (CRITICAL) - ✓ FIXED
───────────────────────────────────────────────────────────────────────────────
IMPACT: Frontend cannot call API at all
FILE: .env (line 54)

BEFORE:  ALLOWED_ORIGINS=["http://localhost:3000"]
AFTER:   ALLOWED_ORIGINS=http://localhost:3000

WHY IT FAILED:
  The config parser splits on commas, so JSON literal ["url"] became 
  a list with one element: '["http://localhost:3000"]' (the JSON string itself!)
  CORS middleware tries to match this literal string in Origin headers, always fails.

WHY FIX WORKS:
  Plain comma-separated format now correctly parses to: ['http://localhost:3000']
  CORS middleware can now match incoming Origin headers against this URL.

BUG #2: MULTI-FILE UPLOAD LOSES DATA (MEDIUM) - ✓ FIXED
───────────────────────────────────────────────────────────────────────────────
IMPACT: Files silently dropped when file_types count < files count
FILE: apps/api/routers/rfp.py (lines 108-127)

SCENARIO THAT FAILS:
  User uploads: [file1.docx, file2.pdf, file3.txt]
  But provides: file_types="MAIN,ANNEX"
  
  Result: Only first 2 files stored, file3.txt is SILENTLY LOST!
  No error message to user.

BEFORE:
  type_list = [t.strip() for t in file_types.split(",")]
  for idx, (file, ftype) in enumerate(zip(files, type_list, strict=False)):
      # Loop only runs 2 times, file3 never processed!

AFTER:
  type_list = [t.strip() for t in file_types.split(",")]
  
  # NEW: Validate file types match number of files
  if len(files) != len(type_list):
      raise HTTPException(
          status_code=status.HTTP_400_BAD_REQUEST,
          detail=f"File type mismatch: {len(files)} files provided but {len(type_list)} types specified."
      )
  
  for idx, (file, ftype) in enumerate(zip(files, type_list)):

WHY FIX WORKS:
  Now catches the mismatch and returns clear 400 error to user.
  User knows exactly how to fix it (provide matching number of types).
