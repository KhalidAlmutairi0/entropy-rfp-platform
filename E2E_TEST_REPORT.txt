[BUG #1] CORS Configuration Error (Severity: CRITICAL)
─────────────────────────────────────────────────────────────────────────────
File: .env
Line: 54
Issue: ALLOWED_ORIGINS format is a JSON literal, not comma-separated string

Current (WRONG):
  ALLOWED_ORIGINS=["http://localhost:3000"]

Expected (CORRECT):
  ALLOWED_ORIGINS=http://localhost:3000

Impact: 
  - CORS will reject all web app requests
  - Frontend will get CORS errors when calling API
  - Login and all API calls will fail with 403 Forbidden

Explanation:
  The validator in config.py (line 75) splits on commas:
    return [origin.strip() for origin in v.split(",")]
  
  When given '["http://localhost:3000"]', it produces:
    ['["http://localhost:3000"]']  ← literal JSON string, not the URL!
  
  FastAPI CORS middleware then tries to match the header value against 
  the literal string '["http://localhost:3000"]', which never matches.

Fix: Change .env line 54 from:
  ALLOWED_ORIGINS=["http://localhost:3000"]
To:
  ALLOWED_ORIGINS=http://localhost:3000
